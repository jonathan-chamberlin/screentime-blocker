# Privacy Disclosure Answers (Draft)

Use this as a draft when filling Chrome Web Store privacy and data-use fields.
Always verify fields against the current dashboard wording.

## Data Collection Summary

Brainrot Blocker processes user settings and browsing context to provide site blocking and focus session functionality.

## Data Types Potentially Processed

1. User settings and extension state
- blocked sites
- allowed paths
- productive site/app lists
- session preferences and timer state

2. Optional account data (if signed in)
- identity token
- display name
- profile image URL

3. Optional activity metrics (if backend sync is enabled)
- session start/end
- minutes completed
- blocked-attempt counts

4. Optional reminder note text
- free-form reminder field entered by the user

## Data Not Requested

- card numbers
- bank account numbers
- payment credentials

## Data Use

- core extension functionality (blocking, timers, configuration)
- optional leaderboard display
- optional cross-device configuration sync

## Data Sharing

- no sale of user data
- no ad targeting based on user data
- data is shared only with service providers required for core auth/backend operation

## User Controls

- users can use extension-only mode without companion app
- users can sign out to disable account-linked features
- users can clear local data by removing extension data
